# 阈值调参与实验指标（智能安防系统）

本系统以 `insightface` 提取 512 维 face embedding，并使用余弦相似度（embedding 已归一化，余弦相似度等价于点积）进行匹配。人脸落入 ROI 矩形区域后，计算相似度：

- `score = dot(embedding_query, embedding_db)`
- 若 `score >= threshold`，则认为该人脸在名单中。

## 1. 实验目的

1. 评估不同 `threshold` 对黑名单告警的影响（召回率/误报率）。
2. 对比“全画面识别（无ROI限制）”与“ROI 限制识别”的性能与误报差异。
3. 统计系统处理性能（FPS、平均事件触发延迟/耗时）。

## 2. 数据与标注建议

- 选择若干段视频（至少 3 段），包含以下情况：
  - 黑名单人员出现（至少若干次）
  - 白名单人员出现（若系统要求对其进行记录/提示）
  - 同场景中的未知人员出现
  - 不同光照/角度/遮挡条件
- 为每段视频人工标注“黑名单命中时刻”（可用帧号或时间戳）。

## 3. 指标口径（TP/FP/FN）

对某个视频段在给定 `threshold` 与策略（ROI或全画面）下，统计以下集合：

- **TP（True Positive）**：系统产生 `blacklist_match` 告警，且时间戳落在人工标注命中时间允许窗口内（例如 +/- 1 秒或 +/- N 帧）。
- **FP（False Positive）**：系统产生告警，但人工未标注为黑名单命中。
- **FN（False Negative）**：人工标注黑名单命中，但系统未产生告警。
- **TN（True Negative）**：通常不在视频逐帧场景中显式统计，可用“非告警帧 vs 命中帧”方式扩展，但论文里可优先报告 TP/FP/FN。

基于以上口径计算：

- **Precision（精确率）** = `TP / (TP + FP)`
- **Recall（召回率）** = `TP / (TP + FN)`
- **F1-score** = `2 * Precision * Recall / (Precision + Recall)`
- **误报率（False Alarm Rate）** 可报告 = `FP / (TP + FP)`（与 Precision 同义）

> 建议在论文中明确你采用的“时间匹配窗口”，例如 `±1s`，并固定窗口用于所有实验对比。

## 4. 阈值调参流程

1. 固定 ROI 策略（建议从你系统默认的 ROI 配置开始）。
2. 选取阈值候选集合，例如：`[0.08, 0.10, 0.12, 0.14, 0.16, 0.18]`（可按你模型分布微调步长）。
3. 对每个阈值，运行识别任务并导出告警事件（系统会把事件写入 `recognition_events`，前端也可实时查看）。
4. 依据人工标注计算 TP/FP/FN，得到 Precision/Recall/F1。
5. 选择一个综合表现最好的阈值（或在召回率满足最低要求的前提下选择 Precision 最高的阈值）。

## 5. ROI 对比实验设计

### 5.1 策略对比

- 策略 A：ROI 限制（系统默认，ROI 为矩形）
- 策略 B：全画面（等价于 ROI 取整个画面：`x1=0, y1=0, x2=1, y2=1`）

### 5.2 评估项

1. 误报情况：FP、Precision、Recall
2. 告警数量与事件密度（例如同一视频中告警的数量）
3. 性能：平均 FPS、单帧平均耗时（可从识别服务日志或基准脚本读取）

## 6. 性能统计口径

- 处理性能建议报告：
  - 平均处理帧率 FPS
  - 处理前 N 帧的平均 embedding 检测耗时（可用“累计耗时 / N”）
  - 单次告警事件触发后的总耗时（如果你记录生成事件的耗时）

## 7. 输出记录表（论文附表建议）

建议在论文里做表格记录：

- Video ID / 条件（光照、遮挡）
- threshold
- ROI策略（ROI或全画面）
- TP, FP, FN
- Precision, Recall, F1
- 平均 FPS
