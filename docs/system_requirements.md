# 系统需求（智能安防系统）

## 1. 功能需求

### 1.1 权限登录（RBAC）

- 用户可通过账号密码登录，系统使用 JWT 校验身份。
- 支持角色：管理员（ADMIN）、操作员（OPERATOR）、查看者（VIEWER）。
- 管理员具备样本上传、视频资产管理能力；操作员具备 ROI 配置与启动识别任务能力。

### 1.2 多路视频回放展示

- 系统支持展示多个视频资产（可用于论文演示，替代实时摄像头流）。
- 操作员选择其中一路视频进行 ROI 配置。

### 1.3 人脸名单（黑名单/白名单）与告警

- 管理员上传黑名单/白名单人员样本图片。
- 系统提取样本人脸 embedding 并写入数据库。
- 识别服务对 ROI 内人脸进行 embedding 匹配：
  - 命中黑名单：生成 `blacklist_match` 告警事件，并在 Web 端实时提醒。
  - 命中白名单：生成 `whitelist_match` 记录事件（是否告警可在论文中说明策略）。

### 1.4 WebSocket 实时告警推送

- 告警事件写入数据库后，通过 WebSocket(STOMP) 推送前端告警列表。

### 1.5 ROI 矩形标注

- 支持在视频画面上拖拽绘制矩形 ROI。
- ROI 坐标采用归一化方式保存（避免不同分辨率导致偏移）。

## 2. 性能需求（论文写作建议）

- 给出识别服务处理性能：平均 FPS、平均耗时、抽帧策略说明。
- 给出准确性指标：阈值调参下的 Precision/Recall/F1，并给出 ROI 限制策略对误报率的影响。

## 3. 可靠性需求（论文写作建议）

- 对识别任务失败（视频无法打开、embedding库为空、网络不可达）给出状态更新与错误提示。
